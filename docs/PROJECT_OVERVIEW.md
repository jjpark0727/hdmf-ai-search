# AI서치 프로젝트 개요

## 서비스 개요 및 목표

### 비즈니스 배경
- **문제**: 
   - 당사 임직원들이 하이미 통합검색으로 원하는 사내 정보를 바로 찾기가 어려운 상태임 
   - 또한, 사내 정보/문서에 생성형 AI 사용이 불가한 상태이므로, 사내외 정보를 검색 및 정리하는데 많은 시간이 소요됨
- **현황**: 
   - 하이미 통합검색은 거의 사용하지 않고 있으며, 각자의 방법으로 정보를 찾아서 저장 및 활용 중임
   - 사내 문서의 경우, 수작업으로 자료를 검토하고 산출물을 작성 중임
- **목표**: 생성형 AI 및 Agent 기반 서비스로 업무 효율성 향상

### 단계별 개발 계획
- **1단계** (2026.10월 오픈 목표): **사용자가 업로드한 문서**를 기반으로 질의응답, 요약, 번역, 당사 양식에 맞는 보고서 생성 기능 제공.
- **2단계** (2027.5월 오픈 목표): **사내 공유문서 (하이미, 하이드라이브)를 연동**하여, 사용자가 문서를 업로드하지 않아도 질의응답, 출처 문서 접속 링크, 문서 요약 정보 제공. 추후 하이미 통합검색 결과창과 API 연동 필요.
- **3단계** (2027.10월 오픈 목표): **실시간 웹 검색** 기반 질의응답, 요약, 번역, 트렌드 조사, 보고서 생성 기능 등 제공. 


### 주요 사용자
- 당사 임직원

## 필요 기술 및 개발 관련

### 기술 스택

| 영역 | 기술 | 담당 |
|------|------|------|
| Agent Framework | LangChain + LangGraph | 데이터사이언스파트(DA) |
| Vector DB | 미정 | 데이터사이언스파트(DA) |
| Embedding Model | BGE-M3 (변동가능) | 데이터사이언스파트(DA) |
| LLM Model | Qwen3 | 데이터사이언스파트(DA) |
| Frontend | 미정 | 데이터사이언스파트(DA) + 디지털기획파트 (박소희 대리) |
| Backend | FastAPI (변동가능) | 데이터사이언스파트(MLE) |
| System | - | 서비스개발실 or HDS (기존 시스템 연동 필요 시) |


### 서비스 구축 방향
- 구축 위치: 내부망에서 접속 가능한 AI Portal(가칭) 링크에 접속하여, AI서치(가칭) 기능에 접근. AI Portal의 경우, 업무시스템링크 등 **업무PC에 최초 로그인 시 바로 접속이 가능한 형태**로 제공 필요.
- LLM 모델은 파인튜닝이 필요하며, 임베딩 모델의 경우 필요 시 파인튜닝 진행

### 개발 환경
- GPU 구매 일정 이슈로 인해, 1단계 서비스의 경우 기구축된 분석계(A100) 자원 활용하여 개발 (및 운영) 필요하며, 현재 MLE (이윤재 대리) 측에서 KT DS와 분석계 활용을 위한 환경 세팅 진행 중

### 분석계 신청 방법 
현재 분석계는 개인정보보호 정책 상 중요단말기(별도PC 지급) 내에서만 사용이 가능하며, 중요단말기는 유형자산관리시스템 통해 신청 가능합니다.(상세 신청 방법은 쪽지로 전달 예정)

1. 분석계 방화벽 신청 (담당자: 정보보호 파트)
2. 정보포탈 내 분석계 계정 신청 (담당자: 데이터사이언스파트)
3. 분석계 로그인 (SSO 연동, 중요 단말기에서 로그인)
